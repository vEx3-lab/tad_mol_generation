# SELFIES 分子生成与强化学习优化项目

## 1. 项目简介

本项目面向候选药物分子生成与优化，核心目标是使用深度生成模型生成化学有效分子，并结合多目标奖励函数和强化学习方法，引导模型生成 docking 表现更好、药物相似性更高、合成可行性更好的分子。

当前主线方案：

```text
SMILES/SELFIES 数据
  -> SELFIES 词表构建
  -> Decoder-only Transformer 预训练
  -> SELFIES 自回归生成
  -> SELFIES 解码为 SMILES
  -> RDKit / Open Babel / AutoDock Vina 评分
  -> Vina + QED + SA + LogP 多目标 reward
  -> GSPO / REINFORCE 强化学习微调
  -> 生成结果评估与模型保存
```

相比直接使用 SMILES，SELFIES 的优势是生成字符串更容易保持化学有效性，适合分子生成任务。

## 5. 预训练

运行：

```powershell
python main_selfies.py
```

预训练流程：

1. 读取 `config/decoder_only_tfm_config.yaml`。
2. 加载 `data/htvs_molecules_with_selfies.csv`。
3. 构建 `selfies_vocab`。
4. 保存 vocab 到：

```text
model/decoder_only_tfm/vocab.json
```

5. 使用 K-Fold 训练 Decoder-only Transformer。
6. 保存每个 fold 的最佳模型和 loss 曲线。

重要说明：

后续生成、微调、RL 都应复用同一个 `vocab.json`，避免 checkpoint 与 token id 映射不一致。


## 6. 微调

## 14. 常见问题

### 1. 为什么要保存 vocab.json？

因为模型 checkpoint 的 embedding 和输出层都依赖 token id。如果每次从 CSV 重新构建 vocab，token id 可能变化，导致模型权重与生成逻辑不一致。


### 2. 为什么要 canonicalize SMILES？

同一个分子可以有多种 SMILES 写法。canonicalize 后才能准确去重、计算 novelty、复用 Vina cache。


### 3. 为什么 Vina 要缓存？

Vina 是 RL 流程中最慢的 oracle。RL 训练会重复生成相同或相似分子，缓存能显著减少重复 docking。


### 4. 为什么没有自动归档历史脚本？

当前仓库已有大量未提交和未跟踪实验文件。为了避免误伤已有结果，本轮只改主线文件和新增工具，不移动历史实验脚本。
